
🔴다음 사양에 맞게 팀 공지사항 목록 페이지를 React + shadcn/ui로 구현해줘.

---

## 역할
이 컴포넌트는 NoticeListPage.jsx (또는 .tsx) 이며,
전체 공지사항 앱의 3페이지 중 첫 번째 페이지다.
나머지 두 페이지(NoticeFormPage, NoticeDetailPage)는 별도 파일로 분리 예정이므로,
라우팅 연결 준비만 해두고 실제 구현은 포함하지 않는다.

---

## 기술 스택
- React (functional component + hooks)
- shadcn/ui 컴포넌트 사용
- Tailwind CSS
- 아이콘: lucide-react

---

## 데이터 구조 (공유 타입 — 나머지 페이지와 동일하게 사용)
각 공지사항 객체는 다음 필드를 가진다:

{
  id: string,           // nanoid 등 고유 ID
  title: string,        // 제목
  content: string,      // 본문 (Markdown 원문 string)
  author: string,       // 작성자
  createdAt: string,    // ISO 8601 날짜 문자열
  isPinned: boolean,    // 상단 고정 여부
  category: string      // 카테고리 (예: "공지" | "업무" | "기타")
}

샘플 더미 데이터 3~5개를 initialNotices 상수로 파일 내 선언해둔다.

---

## 상태 관리
- notices: 공지 배열 (useState, 초기값은 initialNotices)
- searchQuery: 검색어 문자열 (useState)
- selectedCategory: 선택된 카테고리 필터 (useState, 기본값 "전체")

## UI 구성

### 헤더 영역
- 페이지 타이틀: "📢 팀 공지사항"
- 우측에 새 공지 작성 버튼 (PenSquare 아이콘 + "새 공지 작성" 텍스트)
  → onClick 시 onNavigate("form") 콜백 호출 (부모에서 주입)



### 필터/검색 영역 (헤더 아래)
- 검색 인풋: placeholder "제목 또는 작성자 검색..."
  → searchQuery state와 양방향 바인딩
- 카테고리 필터 탭 또는 Badge 버튼: ["전체", "공지", "업무", "기타"]
  → 선택된 탭은 primary 색상으로 강조

### 공지 카드 목록
- 필터링 로직:
  1. selectedCategory !== "전체" 이면 category 일치하는 것만
  2. searchQuery 있으면 title 또는 author에 포함된 것만
  3. isPinned === true 인 항목을 항상 최상단 정렬
- 각 카드 (shadcn Card 컴포넌트 사용):
  - 상단 고정 배지: isPinned일 때 "📌 고정" Badge 표시
  - 카테고리 Badge (category 값)
  - 제목 (font-semibold, 클릭 시 onNavigate("detail", http://notice.id) 호출)
  - 본문 미리보기: content를 앞 80자만 잘라서 "..." 처리
  - 하단: 작성자 (User 아이콘), 날짜 (Calendar 아이콘), 삭제 버튼 (Trash2 아이콘)
  - 삭제 클릭 시 해당 id를 notices에서 제거, 확인 dialog 없이 즉시 삭제
- 카드 hover 시 shadow 강조 (transition 포함)

## 🔧 수정 사항: 새 공지 작성 — 페이지 전환 → 모달 팝업으로 변경

### 1. State 추가
컴포넌트 상단 useState 목록에 아래 3개를 추가한다:

- isModalOpen: boolean — 모달 열림/닫힘 제어
- formData: object — { title, author, category, content, isPinned }
  - 초기값: { title: "", author: "", category: "일반", content: "", isPinned: false }
- errors: object — { title, author, content }
  - 초기값: {}

---

### 2. 헤더 버튼 수정
"새 공지 작성" 버튼의 onClick을 아래와 같이 변경한다:

- 기존: onNavigate("form") 호출
- 변경: isModalOpen을 true로 설정

---

### 3. 모달 컴포넌트 추가
return문 최상위 wrapper div 안, 목록 JSX 아래에 모달 블록을 추가한다.

#### 모달 조건부 렌더링
- isModalOpen이 true일 때만 렌더링

#### 오버레이
- 화면 전체를 덮는 fixed 레이어
- 배경: 반투명 검정 (bg-black/50)
- backdrop-blur 적용
- 클릭 시 isModalOpen을 false로 설정, formData 및 errors 초기화

#### 모달 박스 (오버레이 위에 중앙 배치)
- 위치: fixed, 화면 정중앙 (top-50% left-50%, translate -50% -50%)
- 크기: width 66vw / height 75vh
  - min-width: 480px
  - max-width: 860px
- 배경: 카드 배경색 (테마와 동일)
- border-radius, shadow-xl 적용
- 내부 레이아웃: flex-col (헤더 / 폼 스크롤 영역 / 푸터 고정)
- 오버레이 클릭 이벤트가 모달 박스까지 전파되지 않도록 stopPropagation 처리

#### 모달 헤더
- 좌측: "📝 새 공지 작성" 타이틀 텍스트
- 우측: X 아이콘 버튼 → 클릭 시 모달 닫기 + formData/errors 초기화
- 하단 border로 구분선 표시

#### 폼 필드 (스크롤 가능 영역, flex-1 + overflow-y-auto)
아래 순서로 배치, 각 필드 사이 간격 유지:

1. 제목 (Input)
   - label: "제목 *"
   - placeholder: "공지 제목을 입력하세요"
   - errors.title 존재 시 border-red + 에러 메시지 표시

2. 카테고리 (Select)
   - label: "카테고리"
   - 옵션: 일반 / 긴급 / 업무 / 공지

3. 작성자 (Input)
   - label: "작성자 *"
   - placeholder: "작성자 이름"
   - errors.author 존재 시 border-red + 에러 메시지 표시

4. 상단 고정 (Switch 또는 Checkbox)
   - label: "📌 상단 고정"
   - formData.isPinned 토글

5. 내용 (Textarea)
   - label: "내용 *"
   - rows: 6
   - placeholder: "공지 내용을 입력하세요"
   - errors.content 존재 시 border-red + 에러 메시지 표시

#### 유효성 검사 함수
등록 버튼 클릭 시 실행:
- title가 빈 문자열이면 errors.title = "제목을 입력해주세요"
- author가 빈 문자열이면 errors.author = "작성자를 입력해주세요"
- content가 빈 문자열이면 errors.content = "내용을 입력해주세요"
- 에러가 하나라도 있으면 제출 중단

#### 등록 처리 (유효성 통과 시)
notices 배열에 아래 객체를 추가한다:
- id: Date.now()
- title: formData.title
- content: formData.content
- author: formData.author
- category: formData.category
- createdAt: new Date().toISOString()
- isPinned: formData.isPinned

이후 isModalOpen을 false, formData 및 errors를 초기값으로 초기화

#### 모달 푸터 (하단 고정, 상단 border 구분선)
- 우측 정렬, 두 버튼 나란히 배치
- 취소 버튼 (outline variant): 클릭 시 모달 닫기 + 초기화
- 등록 버튼 (default variant, PenSquare 아이콘): 클릭 시 유효성 검사 → 등록 처리

#### 모달 애니메이션
- 진입: opacity 0→1, scale 0.95→1 (duration 200ms ease-out)
- 퇴장: opacity 1→0, scale 1→0.95 (duration 150ms)
- Tailwind transition 클래스 활용

## Props 인터페이스
NoticeListPage는 다음 props를 받는다:

- notices: Notice[]                        — 공지 목록
- onDelete: (id: string) => void           — 삭제 핸들러
- onNavigate: (page: string, id?: string) => void  — 페이지 이동 콜백

단, 이 파일 단독 실행/미리보기용으로 props 없이도 동작하도록
initialNotices를 기본값으로 사용하는 내부 fallback useState도 함께 제공한다.
