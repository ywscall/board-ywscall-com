
🔴다음 사양에 맞게 팀 공지사항 목록 페이지를 React + shadcn/ui로 구현해줘.

---

## 역할
이 컴포넌트는 NoticeListPage.jsx (또는 .tsx) 이며,
전체 공지사항 앱의 3페이지 중 첫 번째 페이지다.
나머지 두 페이지(NoticeFormPage, NoticeDetailPage)는 별도 파일로 분리 예정이므로,
라우팅 연결 준비만 해두고 실제 구현은 포함하지 않는다.

---

## 기술 스택
- React (functional component + hooks)
- shadcn/ui 컴포넌트 사용
- Tailwind CSS
- 아이콘: lucide-react

---

## 데이터 구조 (공유 타입 — 나머지 페이지와 동일하게 사용)
각 공지사항 객체는 다음 필드를 가진다:

{
  id: string,           // nanoid 등 고유 ID
  title: string,        // 제목
  content: string,      // 본문 (Markdown 원문 string)
  author: string,       // 작성자
  createdAt: string,    // ISO 8601 날짜 문자열
  isPinned: boolean,    // 상단 고정 여부
  category: string      // 카테고리 (예: "공지" | "업무" | "기타")
}

샘플 더미 데이터 3~5개를 initialNotices 상수로 파일 내 선언해둔다.

---

## 상태 관리
- notices: 공지 배열 (useState, 초기값은 initialNotices)
- searchQuery: 검색어 문자열 (useState)
- selectedCategory: 선택된 카테고리 필터 (useState, 기본값 "전체")

## UI 구성

### 헤더 영역
- 페이지 타이틀: "📢 팀 공지사항"
- 우측에 새 공지 작성 버튼 (PenSquare 아이콘 + "새 공지 작성" 텍스트)
  → onClick 시 onNavigate("form") 콜백 호출 (부모에서 주입)

### 필터/검색 영역 (헤더 아래)
- 검색 인풋: placeholder "제목 또는 작성자 검색..."
  → searchQuery state와 양방향 바인딩
- 카테고리 필터 탭 또는 Badge 버튼: ["전체", "공지", "업무", "기타"]
  → 선택된 탭은 primary 색상으로 강조

### 공지 카드 목록
- 필터링 로직:
  1. selectedCategory !== "전체" 이면 category 일치하는 것만
  2. searchQuery 있으면 title 또는 author에 포함된 것만
  3. isPinned === true 인 항목을 항상 최상단 정렬
- 각 카드 (shadcn Card 컴포넌트 사용):
  - 상단 고정 배지: isPinned일 때 "📌 고정" Badge 표시
  - 카테고리 Badge (category 값)
  - 제목 (font-semibold, 클릭 시 onNavigate("detail", http://notice.id) 호출)
  - 본문 미리보기: content를 앞 80자만 잘라서 "..." 처리
  - 하단: 작성자 (User 아이콘), 날짜 (Calendar 아이콘), 삭제 버튼 (Trash2 아이콘)
  - 삭제 클릭 시 해당 id를 notices에서 제거, 확인 dialog 없이 즉시 삭제
- 카드 hover 시 shadow 강조 (transition 포함)

### 빈 상태
- 검색/필터 결과가 0개일 때:
  "검색 결과가 없습니다" 메시지를 중앙 정렬로 표시 (Inbox 아이콘 + 텍스트)

## Props 인터페이스
NoticeListPage는 다음 props를 받는다:

- notices: Notice[]                        — 공지 목록
- onDelete: (id: string) => void           — 삭제 핸들러
- onNavigate: (page: string, id?: string) => void  — 페이지 이동 콜백

단, 이 파일 단독 실행/미리보기용으로 props 없이도 동작하도록
initialNotices를 기본값으로 사용하는 내부 fallback useState도 함께 제공한다.

## 스타일 가이드
- 전체 배경: bg-gray-50
- 카드: bg-white, rounded-xl, border border-gray-100
- 고정 공지 카드: bg-amber-50, border-amber-200
- 새 공지 작성 버튼: variant="default" (primary)
- 삭제 버튼: variant="ghost", text-red-400, hover:text-red-600
- 카테고리 Badge 색상:
  - "공지" → blue
  - "업무" → green
  - "기타" → gray
- 날짜 포맷: http://YYYY.MM.DD (한국식)
